# AGENTS.md

## プロジェクト概要

BowserJRは、Zenn CLIを使用して技術記事を作成・公開するリポジトリである。

- 目的: エビデンスベースの技術記事を効率的に執筆・公開する
- 対象読者: 技術者、研究者、エンジニア
- 記事種類: 技術解説、チュートリアル、研究成果の共有

## コンテンツ品質基準

### 情報の信頼性

情報源は以下の優先順位で選択し、必ず出典を明記すること。

1. 公式ドキュメント - 最優先の情報源
2. 学術論文・査読済み文献 - 理論的背景や研究成果
3. 信頼できる技術ブログ - 実践的な知見
4. GitHub Issue/Discussion - 最新の議論や既知の問題

#### 記載必須項目

- 参照先のURL
- バージョン情報（ライブラリ、フレームワーク、言語など）
- アクセス日時（リンク切れリスクが高い情報の場合）

### エビデンスベースの記述

すべての主張には根拠を示すこと。

- 確定事実: 公式ドキュメントや実装結果に基づく記述
- 推測・見解: 明示的に「推測」「見解」として区別する
- 不確実性: 検証できていない内容は「可能性がある」など曖昧さを明示する

### 専門性と正確性

#### 専門用語

- 初出時に定義を記載する
- 記事内で一貫した用語を使用する
- 必要に応じて補足説明やリンクを追加する

#### 技術的正確性

- コード例は実際に動作確認済みのものを使用する
- 非推奨機能は避け、最新のベストプラクティスに従う
- 将来的に変更される可能性がある内容には注記する

### 読者への配慮

#### 前提知識の明示

記事冒頭で以下を明確にする。

- 理解に必要な前提知識
- 関連する基礎概念へのリンク
- 想定する読者レベル

#### 段階的な説明

- 基礎→応用: 簡単な概念から複雑な内容へ段階的に説明する
- 具体→抽象: 実例を示してから一般化する
- 理論→実践: 概念説明の後に実装例を提供する

#### コード例の提供

- 理論的な説明には必ず実例を添える
- コードは最小限かつ動作可能なものにする
- 重要な部分にはコメントを付ける

## メタデータ要件

### Front Matter 必須項目

記事のFront Matterには以下を必ず含める。

```yaml
---
title: "内容を正確に反映した記事タイトル"
emoji: "📝"  # 記事内容に関連したアイコン
type: "tech"  # tech または idea
topics: ["topic1", "topic2"]  # 関連技術タグ（最大5つ）
published: false  # 公開状態: true（公開）/ false（下書き）
---
```

### タイトルのガイドライン

- 内容を正確に反映させる
- 具体的な技術名やキーワードを含める
- 読者が得られる知識や解決できる問題を示す

### トピックス選定

- 記事の主要な技術・概念を選ぶ
- 一般的な技術名を使用する（検索性を考慮）
- 最大5つまで設定可能

## 品質チェック

記事公開前に以下を確認すること。

### 自動チェック（lint-staged で実行）

- Markdownlint: 構文・スタイルチェック
- Textlint: 文体（である調）・文法チェック

### 手動確認

- [ ] すべての主張に根拠が示されているか
- [ ] 引用・参照が適切に記載されているか
- [ ] コード例が動作するか
- [ ] 論理の飛躍や矛盾がないか
- [ ] 読者が段階的に理解できる構成になっているか
